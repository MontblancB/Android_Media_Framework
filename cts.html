Android Compatibility Test Suite

1 Overview: CTS란?

CTS (Compatibility Test Suite)는 Android 디바이스가 플랫폼 호환성 요구사항을 충족하는지 검증하는 무료 테스트 도구입니다.

🎯 주요 목적

플랫폼 호환성 보장

  • 일관된 앱 동작 보장
  • Android API 정확한 구현 검증
  • 조기 비호환성 발견
  • GMS 인증 필수 요건

📋 호환성 요구사항

Android 호환 디바이스 조건

  • CDD (Compatibility Definition Document) 준수
  • 모든 CTS 테스트 통과
  • User Build에서 실행
  • 다중 ABI 지원 시 모든 ABI 테스트

🔧 테스트 프레임워크

자동화된 테스트 하네스

  • Desktop에서 실행되는 Test Harness
  • 디바이스에서 실행되는 Test Cases
  • JUnit 기반 Java 테스트
  • APK 형태로 패키징

📱 GMS 인증

Google Mobile Services

  • Play Store, Gmail, YouTube 사전 설치
  • CTS 통과 필수
  • 상용 디바이스 필수 요건
  • 사용자 경험 일관성

📊 테스트 유형

🔬 Unit Tests

원자적 코드 단위 검증

  • 단일 클래스 테스트
  • 메서드 동작 검증
  • 에러 처리 확인
  • 격리된 환경에서 실행

⚙️ Functional Tests

API 조합 테스트

  • 상위 레벨 유스케이스
  • API 간 상호작용
  • 통합 시나리오
  • 실제 사용 패턴

💪 Robustness Tests

시스템 내구성 평가

  • 스트레스 테스트
  • 예외 상황 처리
  • 리소스 부족 시나리오
  • 장시간 안정성

⚡ Performance Tests

성능 벤치마크

  • FPS 렌더링 측정
  • 응답 시간 검증
  • 처리량 평가
  • 성능 기준 충족

3 미디어 CTS 테스트

MediaCodec, MediaExtractor, MediaPlayer 등 미디어 프레임워크 컴포넌트에 대한 CTS 테스트를 설명합니다.

🎬 MediaCodec CTS

🎯 코덱 기능 테스트

인코딩/디코딩 검증

  • MediaCodecList의 모든 코덱 테스트
  • 다양한 해상도 (720p, QCIF, QVGA)
  • 프로파일 및 레벨 지원
  • 색상 포맷 처리

⚡ 성능 및 Capabilities

MediaCodecCapabilities 검증

  • 지원 해상도 범위
  • 비트레이트 범위
  • 프레임레이트 범위
  • 성능 클래스 요구사항

🔄 비동기 모드

Callback 기반 처리

  • onInputBufferAvailable
  • onOutputBufferAvailable
  • onError 처리
  • onOutputFormatChanged

📹 테스트 벡터

표준 미디어 파일

  • Big Buck Bunny (8-bit)
  • Cosmos Laundromat (10-bit)
  • 다양한 코덱 및 포맷
  • 자동 다운로드

📦 MediaExtractor CTS

📂 데이터 소스 처리

  • 파일 경로 (File Path)
  • HTTP URL
  • FileDescriptor
  • MediaDataSource

🎵 트랙 선택 및 포맷

  • 트랙 개수 확인
  • MediaFormat 정보 검증
  • MIME 타입 확인
  • 트랙 선택/해제

📊 샘플 추출

  • 샘플 데이터 읽기
  • Presentation Time 검증
  • 트랙 인덱스 확인
  • 플래그 (SYNC, ENCRYPTED)

🔒 DRM 지원

  • PSSH 정보 추출
  • 암호화된 샘플 처리
  • DRM 스킴 확인
  • 보호된 콘텐츠 테스트

▶️ MediaPlayer CTS

🔄 생명주기 관리

  • create() → start() → pause()
  • stop() → reset() → release()
  • 상태 전환 검증
  • 잘못된 상태 처리

🎮 재생 제어

  • start/pause/stop
  • seekTo() 정확도
  • 재생 속도 제어
  • 루핑 기능

🎭 A/V 동기화

  • 립싱크 허용 오차
  • 타임스탬프 정확도
  • 드리프트 보정

📢 이벤트 처리

  • OnCompletionListener

  • OnInfoListener

✅ CTS

Compatibility Test Suite

  • 목적: Android 플랫폼 호환성
  • 대상: Framework APIs, 시스템 동작
  • 언어: Java (JUnit)
  • 실행: APK 형태로 디바이스에서
  • 범위: 앱 레벨 호환성

🔧 VTS

Vendor Test Suite

  • 목적: 벤더 구현 검증
  • 대상: HAL, 커널, VNDK
  • 언어: C++ (GTest), Python
  • 실행: 디바이스 네이티브
  • 범위: 하드웨어 레벨 호환성

📱 GTS

GMS Test Suite

  • 목적: GMS 라이선스 획득
  • 대상: Google 앱 및 서비스
  • 언어: Java, 다양
  • 실행: GMS 앱 테스트
  • 범위: Google 생태계 통합
💡 xTS 통과 필수:
  • 상용 Android 디바이스는 CTS, VTS, GTS 모두 통과해야 GMS 인증 획득
  • CTS는 앱 호환성, VTS는 하드웨어 호환성, GTS는 Google 서비스 통합 검증
  • 각 테스트 스위트는 서로 보완적이며 전체 시스템 품질 보장

📝 인증 단계

1️⃣ CDD 준수

Compatibility Definition Document

  • 소프트웨어 요구사항 구현
  • 하드웨어 기능 구현
  • 성능 기준 충족
  • 보안 요구사항 준수

2️⃣ xTS 실행

테스트 스위트 실행

  • CTS: 플랫폼 호환성
  • VTS: 벤더 구현
  • GTS: GMS 통합
  • 연속적인 자가 테스트

3️⃣ 결과 제출

Google에 리포트 제출

  • XML 테스트 리포트
  • 압축된 결과 디렉토리
  • 디바이스 정보
  • 빌드 정보

4️⃣ GMS 승인

인증 획득

  • Google 검토 및 승인
  • GMS 라이선스 발급
  • Google 앱 사전 설치 권한
  • 상용 출시 가능
✅ 인증 성공 시 혜택:
  • Play Store, Gmail, YouTube 등 Google 앱 사전 설치 가능
  • Google Play Services API 사용 가능
  • Android 브랜드 및 로고 사용 권한
  • 사용자에게 일관된 Android 경험 제공

6 CTS 실행 방법

CTS 테스트를 실행하고 결과를 분석하는 방법을 설명합니다.

💻 환경 준비

필수 요구사항

  • Desktop (Windows/Ubuntu)
  • CTS 패키지 다운로드
  • ADB (Android Debug Bridge)
  • User Build 디바이스

▶️ 실행 명령

CTS 실행

  • ./cts-tradefed: 하네스 시작
  • run cts: 전체 테스트
  • run cts --module [name]: 특정 모듈
  • run cts --plan [name]: 테스트 플랜

📊 결과 분석

리포트 확인

  • XML 리포트 생성
  • 브라우저에서 확인 가능
  • 모듈별 통과/실패 상태
  • 실패 원인 로그

🔄 재테스트

실패 케이스 재실행

  • run retry: 실패 케이스만
  • 문제 수정 후 재실행
  • 부분 테스트 가능
  • 연속 통합 지원
⚠️ 주의사항:
  • CTS는 User Build에서만 실행 (eng/userdebug 빌드 불가)
  • 테스트 실행 시간이 매우 길 수 있음 (수 시간~수일)
  • 디바이스는 충전 상태 유지 필요
  • 네트워크 연결 필요 (테스트 벡터 다운로드)
