1 AAOS Overview

Android Automotive OS is an Android platform specifically designed for vehicle infotainment systems (IVI).

What is AAOS? AAOS is a complete vehicle operating system built on top of standard AOSP with vehicle-specific layers, enabling direct communication with vehicle hardware such as ECUs and CAN bus.

üéØ Key Objectives

  • Power vehicle infotainment systems (IVI)
  • Access and control vehicle data
  • UI/UX optimized for driver safety
  • Multi-user/Multi-zone support

üè≠ Major Manufacturers (2024)

  • Volvo/Polestar: XC40 Recharge, Polestar 2/3 (Full AAOS)
  • GM: Chevrolet, GMC, Cadillac (North America lineup)
  • Ford: Lincoln series, Mustang Mach-E
  • BMW: iDrive 8 (Android 13 based, 7 Series, iX)
  • Honda: Acura models
  • Renault: Megane E-Tech, Austral
  • Nissan/Infiniti: Select EV models

‚öôÔ∏è Key Differentiators

  • Vehicle HAL (VHAL) addition
  • Car Services layer
  • android.car API package
  • Driver distraction prevention guidelines
  • Car UI Library templates

3 Car Services

Car Services acts as a mediator between apps and vehicle hardware, exposing vehicle functions as Android APIs.

üéØ Key Roles

  • Mediator: Brokers between apps and vehicle data
  • Security: Permission-based access control
  • Abstraction: Hides hardware complexity
  • Interoperability: Compatibility across manufacturers

üì¶ Core Services

  • CarPropertyManager: Vehicle property management
  • CarAudioService: Multi-zone audio
  • CarPowerService: Power management
  • CarUserService: Multi-user
  • CarInputService: Input events

üîê Security & Permissions

  • Automotive-specific permission system
  • Sensitive vehicle data protection
  • Per-app access control
  • OEM-exclusive permissions

4 Vehicle HAL (VHAL)

Vehicle Hardware Abstraction Layer is the standard interface between Android and vehicle hardware.

üîå VHAL Concept

  • Vehicle hardware abstraction layer
  • Translator between ECU/CAN bus and Android
  • Exposes vehicle data as Properties
  • AIDL/HIDL based interface

üìä Vehicle Properties

  • Speed: PERF_VEHICLE_SPEED
  • Fuel: FUEL_LEVEL_LOW, FUEL_LEVEL
  • Gear: GEAR_SELECTION
  • HVAC: HVAC_TEMPERATURE_SET
  • Door: DOOR_LOCK, DOOR_POS

‚öôÔ∏è Property Types

  • Access: READ, WRITE, READ_WRITE
  • Change Mode:
    • STATIC: Never changes
    • ON_CHANGE: Notifies on change
    • CONTINUOUS: Periodic updates
