Media Stack Porting Checklist

Step-by-step guide for porting Android media stack to a new SoC/platform

01Porting Overview

Porting the Android media stack to a new hardware platform requires a systematic approach. This guide provides a step-by-step checklist to help SoC vendors and OEM engineers successfully complete media porting.

1.1 Porting Roadmap

1.2 Key Components

Component Responsibility Required Complexity
Codec2 HAL Video/Audio codecs Required High
Audio HAL Audio input/output Required Medium
DRM HAL Content protection Conditional High
Camera HAL Camera input Optional High
Display HAL Display output Required Medium

02Phase 1: Preparation

1
Requirements Analysis and Environment Setup
Estimated time: 1-2 weeks

1.1 Hardware Requirements Review

  • Review HW codec IP specs (supported codecs, resolution, profiles)
  • Review audio DSP specifications
  • Review memory architecture (ION/DMA-BUF support)
  • Review security hardware (TrustZone, TEE)
  • Review display output specifications

1.2 Software Requirements

  • Determine target Android version
  • Review mandatory CDD requirements
  • Check GMS certification requirements
  • Set performance targets (resolution, FPS, latency)

1.3 Development Environment Setup

  • Download AOSP source and set up build environment
  • Design vendor source tree structure
  • Configure build system (Android.bp/mk)
  • Set up CI/CD pipeline

03Phase 2: Codec HAL Implementation

2
Codec2 HAL Implementation
Estimated time: 3-4 weeks

2.1 Basic Structure Implementation

  • Implement C2ComponentStore
  • Implement C2Component base class
  • Implement buffer management (C2Buffer, C2Block)
  • Write HAL service daemon

2.2 Video Decoder Implementation

  • H.264/AVC decoder
  • H.265/HEVC decoder
  • VP9 decoder
  • AV1 decoder (optional)
  • Secure decoder (for DRM)

2.3 Video Encoder Implementation

  • H.264/AVC encoder
  • H.265/HEVC encoder
  • VP9 encoder (optional)

2.4 Audio Codec Implementation

  • AAC decoder/encoder
  • MP3 decoder
  • Opus decoder
  • FLAC decoder

2.5 Configuration Files

  • Write media_codecs.xml
  • Write media_codecs_performance.xml
  • Register VINTF manifest

media_codecs.xml Template

<?xml version="1.0" encoding="utf-8"?>
<MediaCodecs>
    <Settings>
        <Setting name="max-video-encoder-input-buffers" value="9" />
    </Settings>

    <Decoders>
        <!-- Video Decoders -->
        <MediaCodec name="c2.vendor.avc.decoder" type="video/avc" rank="0">
            <Limit name="size" min="16x16" max="4096x2160" />
            <Limit name="alignment" value="2x2" />
            <Limit name="block-size" value="16x16" />
            <Limit name="blocks-per-second" min="1" max="2073600" />
            <Limit name="bitrate" range="1-120000000" />
            <Limit name="concurrent-instances" max="16" />
            <Feature name="adaptive-playback" />
            <Feature name="secure-playback" required="true" />
        </MediaCodec>

        <MediaCodec name="c2.vendor.hevc.decoder" type="video/hevc" rank="0">
            <Limit name="size" min="16x16" max="8192x4320" />
            <Limit name="bitrate" range="1-160000000" />
            <Feature name="adaptive-playback" />
        </MediaCodec>
    </Decoders>

    <Encoders>
        <MediaCodec name="c2.vendor.avc.encoder" type="video/avc" rank="0">
            <Limit name="size" min="16x16" max="4096x2160" />
            <Limit name="bitrate" range="1-80000000" />
            <Feature name="bitrate-modes" value="VBR,CBR,CQ" />
        </MediaCodec>
    </Encoders>
</MediaCodecs>

04Phase 3: Audio HAL Implementation

3
Audio HAL Implementation
Estimated time: 2-3 weeks

3.1 Audio HAL Basic Implementation

  • Implement IDevicesFactory
  • Implement IDevice
  • Implement IStreamOut
  • Implement IStreamIn

3.2 Audio Routing

  • Write audio_policy_configuration.xml
  • Configure output devices (speaker, headphone, HDMI)
  • Configure input devices (microphone)
  • Set volume curves

3.3 Advanced Features

  • Implement Effects HAL (optional)
  • Bluetooth audio integration
  • USB audio support
  • HDMI/ARC audio output

audio_policy_configuration.xml Template

<?xml version="1.0" encoding="UTF-8"?>
<audioPolicyConfiguration version="7.0">
    <modules>
        <module name="primary" halVersion="3.0">
            <mixPorts>
                <mixPort name="primary output"
                         role="source"
                         flags="AUDIO_OUTPUT_FLAG_PRIMARY">
                    <profile name="" format="AUDIO_FORMAT_PCM_16_BIT"
                             samplingRates="48000"
                             channelMasks="AUDIO_CHANNEL_OUT_STEREO"/>
                </mixPort>
            </mixPorts>
            <devicePorts>
                <devicePort tagName="Speaker"
                            type="AUDIO_DEVICE_OUT_SPEAKER"
                            role="sink">
                    <profile name="" format="AUDIO_FORMAT_PCM_16_BIT"
                             samplingRates="48000"
                             channelMasks="AUDIO_CHANNEL_OUT_STEREO"/>
                </devicePort>
            </devicePorts>
            <routes>
                <route type="mix" sink="Speaker"
                       sources="primary output"/>
            </routes>
        </module>
    </modules>
</audioPolicyConfiguration>

05Phase 4: DRM Porting

4
DRM HAL and Widevine Porting
Estimated time: 2-3 weeks

4.1 TEE Environment Setup

  • Port TrustZone/TEE OS
  • Implement Secure World communication interface
  • Implement Keymaster/Keymint HAL

4.2 Widevine Porting

  • Integrate OEMCrypto library
  • Implement DRM HAL service
  • Implement Secure Buffer handling
  • Set up key provisioning system

4.3 Certification

  • Apply for Widevine L1 certification
  • Pass Robustness tests
  • Obtain Production keys
Widevine Level Requirements
Level Decryption Decoding Content
L1 Inside TEE Inside TEE HD/4K
L2 Inside TEE REE SD
L3 REE REE SD

06Phase 5: Integration Testing

5
System Integration Testing
Estimated time: 1-2 weeks

5.1 Functional Testing

  • Basic playback test (local files)
  • Streaming playback test (HLS, DASH)
  • DRM content playback test
  • Recording test
  • Concurrent playback test

5.2 Performance Testing

  • Measure startup latency (< 500ms)
  • Measure frame drop rate (< 0.1%)
  • Measure CPU usage
  • Measure memory usage
  • Measure battery consumption

5.3 Stability Testing

  • Long-duration playback test (8+ hours)
  • Repeated play/stop test
  • Memory leak test
  • Stress test

Test Automation Script

#!/bin/bash
# media_stress_test.sh

ITERATIONS=1000
PASS=0
FAIL=0

for i in $(seq 1 $ITERATIONS); do
    echo "=== Iteration $i / $ITERATIONS ==="

    # Playback test
    adb shell am start -a android.intent.action.VIEW \
        -d "file:///sdcard/test_video.mp4" \
        -t "video/*"
    sleep 5

    # Check playback state
    STATE=$(adb shell dumpsys media.player | grep "state")
    if [[ $STATE == *"PLAYING"* ]]; then
        ((PASS++))
    else
        ((FAIL++))
        adb shell bugreport > "bugreport_$i.zip"
    fi

    # Cleanup
    adb shell am force-stop com.android.gallery3d
    sleep 2
done

echo "=== Results ==="
echo "Pass: $PASS / $ITERATIONS"
echo "Fail: $FAIL / $ITERATIONS"

07Phase 6: CTS/VTS Certification

6
Compatibility Test Pass
Estimated time: 2-3 weeks

6.1 VTS (Vendor Test Suite)

  • VtsHalMediaC2V1_0TargetTest
  • VtsHalAudioV7_0TargetTest
  • VtsHalDrmV1_0TargetTest
  • VtsTrebleVintfTest
  • VtsVndkDependencyTest

6.2 CTS (Compatibility Test Suite)

  • CtsMediaTestCases
  • CtsMediaCodecTestCases
  • CtsMediaDecoderTestCases
  • CtsMediaEncoderTestCases
  • CtsMediaDrmFrameworkTestCases
  • CtsAudioTestCases

6.3 GTS (Google Test Suite)

  • GtsMediaTestCases
  • GtsExoPlayerTestCases
  • GtsYouTubeTestCases

Test Execution Commands

# Run VTS
vts-tradefed run commandAndExit vts \
    --module VtsHalMediaC2V1_0TargetTest

# Run CTS media modules
cts-tradefed run commandAndExit cts \
    --module CtsMediaTestCases \
    --module CtsMediaCodecTestCases

# Retry failed tests
cts-tradefed run commandAndExit retry \
    --retry <session_id>

# Check results
cat /path/to/results/test_result.xml | grep "fail"

08Phase 7: GMS Certification

7
Google Mobile Services Certification
Estimated time: 1-2 weeks

7.1 GMS Requirements

  • 100% CTS/VTS pass
  • GTS pass
  • CDD requirements met
  • Latest Security Patch Level

7.2 Submission Documents

  • CTS/VTS reports
  • Build information (fingerprint)
  • Device specifications
  • Widevine certificate

7.3 Final Verification

  • Google app testing (YouTube, Play Movies)
  • SafetyNet pass
  • Play Protect certification
