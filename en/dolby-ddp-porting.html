Dolby Digital Plus (DDP) Porting Guide

Comprehensive porting guide for integrating Dolby Audio Processing (DAP) and DDP decoding into Android Audio HAL

1 Prerequisites & Licensing

Dolby Digital Plus integration is not just a simple software installation. It requires meeting hardware support, licensing agreements, and Android version-specific HAL requirements.

Licensing Required

To deploy Dolby technology on commercial devices, a licensing agreement with Dolby Laboratories is mandatory. After the agreement, you will receive the Dolby Audio for Android (DAX3) SDK or MS12 library.

1.1 Hardware Requirements

Component Requirement Notes
SoC Dolby-certified DSP or Audio Processor Qualcomm Hexagon, MediaTek, Amlogic, etc.
Efuse Dolby feature activation key provisioning Coordination with SoC vendor required
Audio Output HDMI 2.0+ or S/PDIF Required for bitstream passthrough
Memory Additional 64MB+ (for MS12 engine) More required for multichannel processing

2.1 Data Flow

Compressed Offload vs PCM Decode

Offload Mode: Compressed audio data is passed directly to HAL and decoded by MS12
PCM Mode: Framework decodes first, then PCM is passed to HAL for DAP post-processing only

Mode Data Format Dolby Processing Use Case
Offload AC-3 / E-AC-3 / AC-4 bitstream MS12 Decode + DAP Video playback, low-power scenarios
PCM 16/24-bit PCM DAP only Games, VoIP, general apps
Pass-through Original bitstream None (bypass) AVR/Soundbar connection

3 Audio HAL Implementation

3.1 AIDL HAL Structure (Android 14+)

Starting from Android 14, the AIDL interface is mandatory. Main implementation file structure:

vendor/ └── audio/ └── hal/ ├── Android.bp ├── service.cpp # HAL service entry point ├── Module.cpp # IModule implementation ├── StreamOut.cpp # IStreamOut implementation (core) ├── StreamIn.cpp # IStreamIn implementation └── dolby/ ├── DolbyMs12Wrapper.cpp # MS12 wrapper ├── DolbyMs12Wrapper.h ├── DolbyDapWrapper.cpp # DAP wrapper └── DolbyDapWrapper.h

3.2 IStreamOut Implementation (Core)

In StreamOut.cpp, detect Dolby formats and route to the MS12 engine:

// StreamOut.cpp - AIDL implementation example ndk::ScopedAStatus StreamOut::open( const StreamDescriptor& descriptor, const std::shared_ptr& callback) { // 1. Check audio format AudioFormatDescription format = descriptor.format; // 2. Detect Dolby format if (isDolbyFormat(format)) { mDolbyMode = determineDolbyMode(format); initializeDolbyEngine(mDolbyMode); } // 3. Handle Offload flags if (descriptor.flags & AudioOutputFlags::COMPRESS_OFFLOAD) { mIsOffload = true; setupOffloadPipeline(); } return ndk::ScopedAStatus::ok(); } bool StreamOut::isDolbyFormat(const AudioFormatDescription& format) { return format.encoding == AudioFormatType::AC3 || format.encoding == AudioFormatType::E_AC3 || format.encoding == AudioFormatType::E_AC3_JOC || // Atmos format.encoding == AudioFormatType::AC4; } ndk::ScopedAStatus StreamOut::write( const std::vector& data, int64_t* written) { if (mIsOffload && mDolbyMode != DolbyMode::NONE) { // Pass compressed data to MS12 engine *written = mDolbyWrapper->decodeAndProcess(data.data(), data.size()); } else { // Normal PCM processing (DAP post-processing only) *written = mDolbyWrapper->applyDap(data.data(), data.size()); } return ndk::ScopedAStatus::ok(); }

3.3 MS12 Wrapper Implementation

// DolbyMs12Wrapper.cpp class DolbyMs12Wrapper { public: bool initialize(const DolbyConfig& config) { // Dynamic loading of MS12 library mMs12Handle = dlopen("libdolbyms12.so", RTLD_NOW); if (!mMs12Handle) { ALOGE("Failed to load MS12 library: %s", dlerror()); return false; } // Bind function pointers mMs12Init = (ms12_init_fn)dlsym(mMs12Handle, "ms12_init"); mMs12Decode = (ms12_decode_fn)dlsym(mMs12Handle, "ms12_decode"); mMs12Process = (ms12_process_fn)dlsym(mMs12Handle, "ms12_process"); // Initialize MS12 Ms12InitParams params = { .outputMode = config.outputMode, .maxChannels = config.maxChannels, .sampleRate = config.sampleRate, .drcMode = MS12_DRC_LINE, // Dynamic Range Control .dialogEnhance = true, }; return mMs12Init(¶ms) == MS12_OK; } size_t decodeAndProcess(const uint8_t* input, size_t inputSize) { // 1. Decode Ms12DecodeResult decoded; int ret = mMs12Decode(input, inputSize, &decoded); if (ret != MS12_OK) return 0; // 2. DAP post-processing if (mDapEnabled) { applyDapProcessing(decoded.pcmData, decoded.pcmSize); } // 3. Pass to output buffer return writeToOutput(decoded.pcmData, decoded.pcmSize); } private: void* mMs12Handle = nullptr; bool mDapEnabled = true; };

3.4 Metadata Handling

Dolby Atmos content includes JOC (Joint Object Coding) metadata:

// Metadata extraction and application void StreamOut::handleDolbyMetadata(const AudioMetadata& metadata) { // Check for Dolby Atmos metadata if (metadata.has(AUDIO_METADATA_KEY_ATMOS_PRESENT)) { bool atmosPresent = metadata.get(AUDIO_METADATA_KEY_ATMOS_PRESENT); if (atmosPresent) { // Enable Virtualizer (for headphones/speakers) mDolbyWrapper->enableVirtualizer(true); mDolbyWrapper->setRenderMode(DOLBY_RENDER_HEADPHONE_BINAURAL); } } // Dialog Enhancement level if (metadata.has(AUDIO_METADATA_KEY_DIALOG_LEVEL)) { int dialogLevel = metadata.get(AUDIO_METADATA_KEY_DIALOG_LEVEL); mDolbyWrapper->setDialogEnhancement(dialogLevel); } }

6 Testing & Verification

6.1 Basic Verification Checklist

  • Decoder Registration: Query Dolby decoders from MediaCodecList
  • Offload Operation: Verify AC-3/E-AC-3 content is decoded directly in HAL
  • DAP Effects: Confirm audible changes when switching Dolby Atmos modes (Movie, Music, Game)
  • Passthrough: Verify bitstream is passed as-is when connected to HDMI/S-PDIF
  • Channel Mapping: Verify speaker output positions for 5.1/7.1 content

7 Troubleshooting

7.1 Common Issues

Symptom Cause Solution
No sound MS12 library loading failure Check SELinux policy, verify library path
Offload not working audio_policy_configuration.xml misconfiguration Check mixPort flags and format profiles
Noise/glitches Buffer underrun or sample rate mismatch Adjust buffer size, check resampling settings
Swapped channels Channel mapping error Check MS12 output channel order settings
No DAP effect Effect plugin not loaded Check audio_effects.xml and postprocess settings
Passthrough failure HDMI EDID parsing error Check HDMI connection status and sink supported formats

7.2 Enable Debug Logs

# Enable detailed Dolby logs adb shell setprop persist.vendor.dolby.log.level 7 adb shell setprop persist.vendor.audio.debug 1 # Restart HAL adb shell stop audioserver adb shell start audioserver # Collect logs adb logcat -v time | grep -E "Dolby|MS12|DAP|AudioFlinger" > dolby_debug.log

7.3 Performance Optimization

Recommended Buffer Settings

Offload mode: 32KB ~ 64KB (when low latency not required)
Low-latency mode: 4KB ~ 8KB (for games, etc.)
Pass-through: 2KB ~ 4KB (real-time streaming)
