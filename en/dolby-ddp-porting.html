Dolby Digital Plus (DDP) Porting Guide

Comprehensive guide for integrating Dolby Audio Processing (DAP) and DDP support into Android Audio HAL and Framework.

1 Prerequisites & Licensing

Dolby Digital Plus integration is not just a simple software installation; it requires meeting hardware and licensing requirements. Android 14 and above requires implementation aligned with the AIDL interface as Audio HAL has transitioned to it.

⚠️ Licensing Required

To deploy Dolby technology on commercial devices, a licensing agreement with Dolby Laboratories is mandatory, and you must use the provided Dolby Audio for Android (DAX3) SDK or MS12 library.

  • SoC Support: Dolby-certified DSP or Audio Processor (e.g., Qualcomm Hexagon, MediaTek, Amlogic).
  • Efuse Provisioning: Efuse key injection required at SoC level for Dolby feature activation.
  • Android Version: Android 14+ (AIDL HAL required), Android 10+ (AudioTrack Offload support).

2 Integration Architecture (AIDL)

The core of Dolby integration is placing the Dolby engine (DAP/MS12) within the Audio HAL to perform decoding and post-processing. Starting from Android 14, it communicates with the Framework through the AIDL interface.

graph TD App[Media App / ExoPlayer] --> Framework[Android Audio Framework] Framework -- "IStreamOut.write() (AIDL)" --> HAL[Audio HAL (AIDL)] subgraph AudioHAL [Vendor Audio HAL Implementation] StreamIn[Stream Input] --> DolbyEngine[Dolby Engine (MS12/DAP)] DolbyEngine -- "Decoded PCM" --> PostProc[Post Processing] PostProc -- "Multi-channel PCM" --> ALSA[ALSA / TinyALSA] end

3 Audio HAL Integration Steps

3.1 Library Integration

Place the shared libraries provided by Dolby in /vendor/lib64/ and configure permissions.

# Core Decoding & Processing Engine
/vendor/lib64/libdolbyms12.so
# Hardware Acceleration Support (if applicable)
/vendor/lib64/libHwAudio_dcvdec.so
# Tuning Configuration
/vendor/etc/dolby/dax-default.xml

3.2 HAL Implementation (AIDL)

Detect Dolby formats in the IStreamOut.aidl implementation and route them to the engine.

  • Format Detection: Detect AudioFormat.AC3, AudioFormat.E_AC3, AudioFormat.AC4.
  • Offload Playback: Handle AUDIO_OUTPUT_FLAG_COMPRESS_OFFLOAD flag for compressed audio pass-through.
  • Metadata Handling: Apply stream metadata passed through AudioMetadata to the Dolby engine.

4 Framework Configuration

4.1 audio_policy_configuration.xml

Define route connecting mixPort and devicePort that support Dolby formats.

<!-- MixPort Definition -->
<mixPort name="dolby_direct_output" role="source" flags="AUDIO_OUTPUT_FLAG_DIRECT|AUDIO_OUTPUT_FLAG_COMPRESS_OFFLOAD">
    <profile name="" format="AUDIO_FORMAT_AC3"
             samplingRates="44100,48000" channelMasks="AUDIO_CHANNEL_OUT_5POINT1"/>
    <profile name="" format="AUDIO_FORMAT_E_AC3"
             samplingRates="44100,48000" channelMasks="AUDIO_CHANNEL_OUT_7POINT1"/>
</mixPort>

<!-- Route Definition -->
<routes>
    <route type="mix" sink="HDMI Out"
           sources="dolby_direct_output,primary_output"/>
</routes>

4.2 MediaCodec Configuration

Register Dolby decoders in media_codecs.xml to make them accessible through the MediaCodec API.

<MediaCodec name="c2.dolby.eac3.decoder" type="audio/eac3">
    <Limit name="channel-count" max="8" />
    <Limit name="sample-rate" ranges="32000-48000" />
    <Feature name="secure-playback" required="true" />
</MediaCodec>

5 Verification & Testing

After completing the porting, verify the following items.

  • Decoder Availability: Verify that Dolby decoders are listed in MediaCodecList.
  • Pass-through: Verify that bitstream pass-through works correctly when connected via HDMI/SPDIF.
  • DAP Effects: Verify audible changes when changing Dolby Atmos settings (Movie, Music modes, etc.).
  • CTS Tests: Verify passing of Android CTS Media-related test cases.
