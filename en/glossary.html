Android Media Glossary

Media Framework Terminology Dictionary (Alphabetical)

A B C D E F G H I L M N O P R S T U V W

01A

AAC (Advanced Audio Coding)
A successor audio codec to MP3, providing better sound quality at the same bitrate. MIME type: audio/mp4a-latm. The most widely used audio codec on Android.
Codec Audio MPEG-4
ABR (Adaptive Bitrate Streaming)
A streaming technology that dynamically adjusts video quality based on network bandwidth. HLS, DASH, and SmoothStreaming are representative examples. ExoPlayer's DefaultTrackSelector handles ABR automatically.
Streaming ExoPlayer HLS DASH
ACodec
Android's Native-level codec wrapper that abstracts OMX (OpenMAX) codecs. Replaced by CCodec in Codec 2.0. Part of the stagefright library.
Native OMX Codec
AIDL (Android Interface Definition Language)
An interface definition language for inter-process communication (IPC). Replaces HIDL in HAL 3.0+, and is used by MediaCodec, AudioFlinger, and others.
IPC HAL Binder
Audio Focus
An Android system that manages priority when multiple apps play audio simultaneously. Request with AudioManager.requestAudioFocus(), and receive AUDIOFOCUS_LOSS callback when another app acquires focus.
AudioManager Ducking Policy
AudioFlinger
Android's Native audio server that mixes all audio streams and delivers them to the HAL. Backend for AudioTrack and AudioRecord. Check status with dumpsys media.audio_flinger.
Native Audio HAL Mixing
AVC (Advanced Video Coding)
The official name for H.264. MIME type: video/avc. The most widely used video codec, supported by all Android devices.
H.264 Codec Video

02B

Binder
Android's inter-process communication (IPC) mechanism. Used for communication between Framework and Native Services. Check Binder services with dumpsys and service list commands.
IPC Kernel AIDL
Buffer
A memory area for temporarily storing media data. MediaCodec's ByteBuffer and ExoPlayer's LoadControl manage buffers. Buffering is the process of preloading sufficient data before playback.
MediaCodec ExoPlayer Memory

03C

CCodec
The client wrapper for Codec 2.0. Replaces ACodec (OMX) and communicates with Codec 2.0 HAL through IComponentStore. Default codec on Android 10+.
Codec 2.0 HAL Native
CDD (Compatibility Definition Document)
A document defining policy requirements that Android devices must comply with. Basis for CTS testing, using Requirement IDs in [C-X-Y] format. Example: [C-1-1] = Handheld MUST support H.264 1080p@30fps.
CTS Compatibility Policy
Codec
Short for Coder-Decoder, software or hardware that encodes/decodes media data. Includes H.264, H.265 (video), AAC, Opus (audio), and others.
H.264 AAC MediaCodec
Codec 2.0
The next-generation codec HAL for Android 9+. Replaces OMX and provides a modern C++17-based API. Uses C2Component and C2Work structures.
HAL CCodec Native
CTS (Compatibility Test Suite)
The Android compatibility verification test suite. CtsMediaTestCases tests MediaCodec, MediaPlayer, DRM, and more. Run with the cts-tradefed harness.
Testing CDD VTS

04D

DASH (Dynamic Adaptive Streaming over HTTP)
An ABR streaming protocol standardized by MPEG. Provides segment information through manifest files (.mpd). Play with ExoPlayer's DashMediaSource.
Streaming ABR ExoPlayer
Demuxer
A component that separates (demultiplexes) video/audio streams from container files (MP4, MKV). Android's MediaExtractor serves as the demuxer.
MediaExtractor Container Muxer
DRM (Digital Rights Management)
Technology to prevent illegal copying of digital content. Android supports Widevine, PlayReady, and ClearKey. Manage licenses with the MediaDrm API.
Widevine MediaDrm Security
Ducking
The behavior of reducing (ducking) volume instead of stopping playback when audio focus is lost. Reduce volume to 20% on AUDIOFOCUS_LOSS_TRANSIENT_CAN_DUCK events. Navigation TTS is a typical example.
Audio Focus AudioManager AAOS

05E

ExoPlayer
An open-source media player library developed by Google. More flexible than MediaPlayer and supports DASH/HLS, DRM, and ABR. Now migrated to androidx.media3.
Media3 Player AndroidX

06F

Frame Drop
The phenomenon of skipping certain frames during video playback. Caused by CPU/GPU overload, insufficient buffer, or decoding delays. Monitor with dumpsys SurfaceFlinger.
Performance SurfaceFlinger Rendering
ftrace
Linux kernel event tracing tool. Backend for perfetto and systrace. Can be directly controlled at /sys/kernel/debug/tracing.
Kernel perfetto Debugging

07H

H.264 (AVC)
The most widely used video codec. MIME type: video/avc. Supports Baseline/Main/High Profile and is a required codec for all devices per Android CDD.
Codec AVC MPEG-4
H.265 (HEVC)
Successor to H.264, using 50% less bitrate for the same quality. MIME type: video/hevc. Primarily used for 4K/8K video.
Codec HEVC 4K
HAL (Hardware Abstraction Layer)
The interface layer between Android Framework and hardware drivers. Includes Audio HAL, Codec HAL, and others, defined using HIDL/AIDL.
HIDL AIDL Vendor
HDCP (High-bandwidth Digital Content Protection)
Content protection technology for HDMI/DisplayPort. Widevine L1 content requires HDCP 2.2 or higher for external display output.
DRM HDMI Widevine
HIDL (Hardware Interface Definition Language)
HAL interface definition language for Android 8.0+. Used by Codec 2.0, Audio HAL 6.0, and others. Being replaced by AIDL in Android 11+.
HAL AIDL Treble
HLS (HTTP Live Streaming)
An ABR streaming protocol developed by Apple. Provides segments through manifest files (.m3u8). Play with ExoPlayer's HlsMediaSource.
Streaming ABR Apple

08I

I-Frame (Intra-coded Frame)
A frame in video that can be decoded independently. Seek operations move to the nearest I-Frame. The starting frame of a GOP (Group of Pictures).
Codec Seek GOP
IPC (Inter-Process Communication)
Mechanism for exchanging data between different processes in Android. Includes Binder, Socket, Pipe, and others. The media framework primarily uses Binder.
Binder AIDL HIDL

09L

Latency
Time delay from audio input to output. Android CDD requires less than or equal to 100ms for Automotive devices and less than or equal to 80ms for Handheld. Enable low-latency mode with AudioTrack.PERFORMANCE_MODE_LOW_LATENCY.
Audio Performance CDD

10M

Media3
The AndroidX migration version of ExoPlayer 2.x. Package name changed to androidx.media3.*, with new features like MediaSession integration added.
ExoPlayer AndroidX Migration
MediaCodec
Android's low-level codec API. Directly control encoding/decoding with configure(), start(), queueInputBuffer(), and dequeueOutputBuffer().
Codec Framework Native
MediaExtractor
An Android API that extracts audio/video tracks from container files (MP4, MKV). Read frame data with readSampleData().
Demuxer Container Framework
MediaPlayer
Android's high-level media playback API. Easy to play with setDataSource(), prepare(), and start(), but customization is limited.
Player Framework Legacy
MediaSession
API for sharing playback state and metadata with the system. Enables playback control from lock screen, Android Auto, and Wear OS.
Framework Notification AAOS
MIME Type
Standard string representing media format. Examples: video/avc (H.264), audio/mp4a-latm (AAC). Used in MediaCodec.createDecoderByType().
Codec Format
Muxer
A component that combines (multiplexes) multiple streams into a single container file. Android's MediaMuxer serves as the muxer.
MediaMuxer Container Demuxer

11O

OMX (OpenMAX)
Standard interface for multimedia codecs. Used as codec HAL on Android 8 and below, replaced by Codec 2.0 on Android 9+.
Legacy Codec 2.0 HAL

12P

perfetto
System tracing tool for Android 10+. Traces CPU, GPU, Memory, and Binder IPC. Analyze with the ui.perfetto.dev web UI.
Debugging Profiling ftrace
Performance Class
Media performance grades introduced in Android 12+. Divided into Class 11/12/13/14, each with different codec performance, RAM, and display requirements. Check with Build.VERSION.MEDIA_PERFORMANCE_CLASS.
CDD Performance Android 12+

13R

Renderer
Component that renders media in ExoPlayer. Includes MediaCodecVideoRenderer, MediaCodecAudioRenderer, and custom renderers can be implemented.
ExoPlayer Rendering

14S

SPS/PPS (Sequence/Picture Parameter Set)
Configuration data for H.264/H.265 video streams. Includes resolution, profile, color format, etc. Passed to MediaCodec via MediaFormat.KEY_CSD_0.
H.264 Codec Configuration
SurfaceFlinger
Android's screen compositing service. Combines all Surfaces and sends them to the display. Check frame drops, Vsync, etc. with dumpsys SurfaceFlinger.
Graphics Native Rendering

15T

Track
Individual stream within a media container. Includes video track, audio track, subtitle track, etc. Select with MediaExtractor.selectTrack().
MediaExtractor Container Stream
TrackSelector
Component that selects tracks to play in ExoPlayer. DefaultTrackSelector automatically manages ABR, language, and quality priorities.
ExoPlayer ABR Track

16U

Underrun
Phenomenon where playback is interrupted due to empty audio buffer. Check with underruns counter in dumpsys media.audio_flinger. Caused by insufficient CPU or buffer size.
Audio AudioFlinger Performance

17V

VTS (Vendor Test Suite)
Test suite for verifying HAL implementations. Tests Codec HAL, Audio HAL, and others. Run with the vts-tradefed harness.
Testing HAL CTS

18W

Widevine
Google's DRM solution. Has two security levels: L1 (hardware TEE) and L3 (software). Request licenses with the MediaDrm API.
DRM Security Google
