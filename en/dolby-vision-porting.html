Dolby Vision Porting Guide

Detailed guide for integrating Dolby Vision HDR support into Android Codec2, MediaCodec, and Display HAL.

1 Prerequisites & Hardware

Dolby Vision requires integration across the entire video pipeline (decoder → display). Starting from Android 15, the HDR Headroom Control API has been added to control brightness balance for SDR/HDR mixed content.

⚠️ Hardware Requirements

Dolby Vision cannot be implemented with software alone. Dolby Vision Certified SoC and HDR-capable display panel are essential.

  • Android Version: Android 7.0+ (Playback), Android 13+ (Capture), Android 15+ (HDR Headroom).
  • Video Decoder: Hardware decoder supporting HEVC Main 10 profile and Dolby Vision extensions.
  • Display Pipeline: Support for 10-bit/12-bit color depth processing and Dynamic Metadata transmission.

2 Integration Architecture

Dolby Vision integration is divided into Codec2 component implementation and Display HAL metadata handling.

c2.vendor.dv.decoder] --> HW[HW Decoder] HW -- "YUV P010 + RPU Data" --> Buffer[Graphic Buffer] Buffer --> DisplayHAL[Display HAL / HWC] DisplayHAL -- "Tone Mapping" --> Panel[HDR Display Panel] end

3 Codec2 & MediaCodec Integration

3.1 Codec Registration

Register the Dolby Vision decoder in media_codecs.xml and specify the supported profiles.

<MediaCodec name="c2.vendor.dv.decoder" type="video/dolby-vision">
    <Limit name="size" min="176x144" max="3840x2160" />
    <Limit name="alignment" value="2x2" />
    <Limit name="block-size" value="16x16" />
    <Feature name="adaptive-playback" />
    <Feature name="secure-playback" required="true" />
    <!-- Dolby Vision Profiles -->
    <Limit name="concurrent-instances" max="16" />
</MediaCodec>

3.2 Extractor Support

Dolby Vision content consists of Base Layer, Enhancement Layer, and RPU (Reference Picture Unit) metadata. A Dolby Vision-aware Extractor is required so that MediaExtractor can parse dvcC and dvvv boxes.

4 Display HAL Integration

Dynamic metadata (RPU) extracted from the decoder must be passed to the display processor for tone mapping.

  • HdrCapabilities: Display.getHdrCapabilities() must return HDR_TYPE_DOLBY_VISION.
  • Metadata Propagation: Attach metadata to decoded buffers (GraphicBuffer) and pass to SurfaceFlinger and HWC (Hardware Composer).
  • Color Gamut: Verify support for BT.2020 color gamut and PQ (Perceptual Quantizer) EOTF.
  • Tone Mapping: Integration of Dolby Vision Processing Unit (VPU) that adjusts per-scene brightness according to display panel's maximum brightness (Nits).

4.1 Android 15 HDR Headroom

In Android 15, Display HAL must be able to dynamically adjust brightness ratio during SDR/HDR mixing to support the setDesiredHdrHeadroom API.

5 Verification & Testing

Perform the following procedures for integration verification.

  • Capability Check: Verify that Display.getHdrCapabilities() returns HDR_TYPE_DOLBY_VISION when called.
  • Visual Inspection: Verify HDR mark display and brightness/color changes when playing Dolby Vision demo content.
  • Capture Test (Android 13+): Verify that 10-bit recording via Camera2 API is encoded as Dolby Vision Profile 8.4.
