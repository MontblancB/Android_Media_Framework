Dolby Digital Plus (DDP) Porting Guide

Comprehensive guide for integrating Dolby Audio Processing (DAP) and DDP support into Android Audio HAL and Framework.

1 Prerequisites & Licensing

Dolby Digital Plus 통합은 단순한 소프트웨어 설치가 아니며, 하드웨어 및 라이선스 요구사항을 충족해야 합니다. Android 14 이상에서는 Audio HAL이 AIDL 인터페이스로 전환되었으므로 이에 맞춘 구현이 필요합니다.

⚠️ Licensing Required

Dolby 기술을 상용 기기에 탑재하려면 반드시 Dolby Laboratories와의 라이선스 계약이 필요하며, 제공되는 Dolby Audio for Android (DAX3) SDK 또는 MS12 라이브러리를 사용해야 합니다.

  • SoC Support: Dolby 인증을 받은 DSP 또는 Audio Processor (e.g., Qualcomm Hexagon, MediaTek, Amlogic).
  • Efuse Provisioning: SoC 레벨에서 Dolby 기능 활성화를 위한 Efuse 키 주입 필요.
  • Android Version: Android 14+ (AIDL HAL 필수), Android 10+ (AudioTrack Offload 지원).

2 Integration Architecture (AIDL)

Dolby 통합의 핵심은 Audio HAL 내에 Dolby 엔진(DAP/MS12)을 배치하여 디코딩과 후처리(Post-processing)를 수행하는 것입니다. Android 14부터는 AIDL 인터페이스를 통해 Framework와 통신합니다.

3 Audio HAL Integration Steps

3.1 Library Integration

Dolby에서 제공하는 공유 라이브러리를 /vendor/lib64/에 배치하고 권한을 설정합니다.

# Core Decoding & Processing Engine
/vendor/lib64/libdolbyms12.so
# Hardware Acceleration Support (if applicable)
/vendor/lib64/libHwAudio_dcvdec.so
# Tuning Configuration
/vendor/etc/dolby/dax-default.xml

3.2 HAL Implementation (AIDL)

IStreamOut.aidl 구현체에서 Dolby 포맷을 감지하고 엔진으로 라우팅합니다.

  • Format Detection: AudioFormat.AC3, AudioFormat.E_AC3, AudioFormat.AC4 감지.
  • Offload Playback: 압축 오디오 패스스루를 위해 AUDIO_OUTPUT_FLAG_COMPRESS_OFFLOAD 플래그 처리.
  • Metadata Handling: AudioMetadata를 통해 전달된 스트림 메타데이터를 Dolby 엔진에 적용.

4 Framework Configuration

4.1 audio_policy_configuration.xml

Dolby 포맷을 지원하는 mixPort와 devicePort를 연결하는 route를 정의해야 합니다.

<!-- MixPort Definition -->
<mixPort name="dolby_direct_output" role="source" flags="AUDIO_OUTPUT_FLAG_DIRECT|AUDIO_OUTPUT_FLAG_COMPRESS_OFFLOAD">
    <profile name="" format="AUDIO_FORMAT_AC3"
             samplingRates="44100,48000" channelMasks="AUDIO_CHANNEL_OUT_5POINT1"/>
    <profile name="" format="AUDIO_FORMAT_E_AC3"
             samplingRates="44100,48000" channelMasks="AUDIO_CHANNEL_OUT_7POINT1"/>
</mixPort>

<!-- Route Definition -->
<routes>
    <route type="mix" sink="HDMI Out"
           sources="dolby_direct_output,primary_output"/>
</routes>

4.2 MediaCodec Configuration

media_codecs.xml에 Dolby 디코더를 등록하여 MediaCodec API를 통해 접근 가능하게 합니다.

<MediaCodec name="c2.dolby.eac3.decoder" type="audio/eac3">
    <Limit name="channel-count" max="8" />
    <Limit name="sample-rate" ranges="32000-48000" />
    <Feature name="secure-playback" required="true" />
</MediaCodec>

5 Verification & Testing

포팅 완료 후 다음 항목을 검증해야 합니다.

  • Decoder Availability: MediaCodecList에서 Dolby 디코더가 조회되는지 확인.
  • Pass-through: HDMI/SPDIF 연결 시 비트스트림 패스스루가 정상 동작하는지 확인.
  • DAP Effects: Dolby Atmos 설정(Movie, Music 모드 등) 변경 시 청감상 변화가 있는지 확인.
  • CTS Tests: Android CTS의 Media 관련 테스트 케이스 통과 여부 확인.
