Dolby Digital Plus (DDP) Porting Guide

Android Audio HAL에 Dolby Audio Processing (DAP) 및 DDP 디코딩을 통합하기 위한 상세 포팅 가이드

1 사전 요구사항 & 라이선스

Dolby Digital Plus 통합은 단순한 소프트웨어 설치가 아닙니다. 하드웨어 지원, 라이선스 계약, 그리고 Android 버전별 HAL 요구사항을 충족해야 합니다.

Licensing Required

Dolby 기술을 상용 기기에 탑재하려면 Dolby Laboratories와의 라이선스 계약이 필수입니다. 계약 후 Dolby Audio for Android (DAX3) SDK 또는 MS12 라이브러리를 제공받습니다.

2.1 데이터 플로우

Compressed Offload vs PCM Decode

Offload 모드: 압축 오디오 데이터가 직접 HAL로 전달되어 MS12에서 디코딩
PCM 모드: Framework에서 디코딩 후 PCM이 HAL로 전달, DAP 후처리만 수행

모드 데이터 형식 Dolby 처리 사용 시나리오
Offload AC-3 / E-AC-3 / AC-4 비트스트림 MS12 Decode + DAP 영상 재생, 저전력 필요 시
PCM 16/24-bit PCM DAP만 수행 게임, VoIP, 일반 앱
Pass-through 원본 비트스트림 처리 안함 (바이패스) AVR/Soundbar 연결 시

3 Audio HAL 구현

3.1 AIDL HAL 구조 (Android 14+)

Android 14부터는 AIDL 인터페이스가 필수입니다. 주요 구현 파일 구조:

vendor/ └── audio/ └── hal/ ├── Android.bp ├── service.cpp # HAL 서비스 진입점 ├── Module.cpp # IModule 구현 ├── StreamOut.cpp # IStreamOut 구현 (핵심) ├── StreamIn.cpp # IStreamIn 구현 └── dolby/ ├── DolbyMs12Wrapper.cpp # MS12 래퍼 ├── DolbyMs12Wrapper.h ├── DolbyDapWrapper.cpp # DAP 래퍼 └── DolbyDapWrapper.h

3.2 IStreamOut 구현 (핵심)

StreamOut.cpp에서 Dolby 포맷을 감지하고 MS12 엔진으로 라우팅합니다:

// StreamOut.cpp - AIDL 구현 예시 ndk::ScopedAStatus StreamOut::open( const StreamDescriptor& descriptor, const std::shared_ptr& callback) { // 1. 오디오 포맷 확인 AudioFormatDescription format = descriptor.format; // 2. Dolby 포맷 감지 if (isDolbyFormat(format)) { mDolbyMode = determineDolbyMode(format); initializeDolbyEngine(mDolbyMode); } // 3. Offload 플래그 처리 if (descriptor.flags & AudioOutputFlags::COMPRESS_OFFLOAD) { mIsOffload = true; setupOffloadPipeline(); } return ndk::ScopedAStatus::ok(); } bool StreamOut::isDolbyFormat(const AudioFormatDescription& format) { return format.encoding == AudioFormatType::AC3 || format.encoding == AudioFormatType::E_AC3 || format.encoding == AudioFormatType::E_AC3_JOC || // Atmos format.encoding == AudioFormatType::AC4; } ndk::ScopedAStatus StreamOut::write( const std::vector& data, int64_t* written) { if (mIsOffload && mDolbyMode != DolbyMode::NONE) { // MS12 엔진에 압축 데이터 전달 *written = mDolbyWrapper->decodeAndProcess(data.data(), data.size()); } else { // 일반 PCM 처리 (DAP 후처리만) *written = mDolbyWrapper->applyDap(data.data(), data.size()); } return ndk::ScopedAStatus::ok(); }

3.3 MS12 Wrapper 구현

// DolbyMs12Wrapper.cpp class DolbyMs12Wrapper { public: bool initialize(const DolbyConfig& config) { // MS12 라이브러리 동적 로딩 mMs12Handle = dlopen("libdolbyms12.so", RTLD_NOW); if (!mMs12Handle) { ALOGE("Failed to load MS12 library: %s", dlerror()); return false; } // 함수 포인터 바인딩 mMs12Init = (ms12_init_fn)dlsym(mMs12Handle, "ms12_init"); mMs12Decode = (ms12_decode_fn)dlsym(mMs12Handle, "ms12_decode"); mMs12Process = (ms12_process_fn)dlsym(mMs12Handle, "ms12_process"); // MS12 초기화 Ms12InitParams params = { .outputMode = config.outputMode, .maxChannels = config.maxChannels, .sampleRate = config.sampleRate, .drcMode = MS12_DRC_LINE, // Dynamic Range Control .dialogEnhance = true, }; return mMs12Init(¶ms) == MS12_OK; } size_t decodeAndProcess(const uint8_t* input, size_t inputSize) { // 1. 디코딩 Ms12DecodeResult decoded; int ret = mMs12Decode(input, inputSize, &decoded); if (ret != MS12_OK) return 0; // 2. DAP 후처리 if (mDapEnabled) { applyDapProcessing(decoded.pcmData, decoded.pcmSize); } // 3. 출력 버퍼로 전달 return writeToOutput(decoded.pcmData, decoded.pcmSize); } private: void* mMs12Handle = nullptr; bool mDapEnabled = true; };

3.4 메타데이터 처리

Dolby Atmos 콘텐츠는 JOC(Joint Object Coding) 메타데이터를 포함합니다:

// 메타데이터 추출 및 적용 void StreamOut::handleDolbyMetadata(const AudioMetadata& metadata) { // Dolby Atmos 메타데이터 확인 if (metadata.has(AUDIO_METADATA_KEY_ATMOS_PRESENT)) { bool atmosPresent = metadata.get(AUDIO_METADATA_KEY_ATMOS_PRESENT); if (atmosPresent) { // Virtualizer 활성화 (헤드폰/스피커용) mDolbyWrapper->enableVirtualizer(true); mDolbyWrapper->setRenderMode(DOLBY_RENDER_HEADPHONE_BINAURAL); } } // Dialog Enhancement 레벨 if (metadata.has(AUDIO_METADATA_KEY_DIALOG_LEVEL)) { int dialogLevel = metadata.get(AUDIO_METADATA_KEY_DIALOG_LEVEL); mDolbyWrapper->setDialogEnhancement(dialogLevel); } }

6 테스트 & 검증

6.1 기본 검증 체크리스트

  • 디코더 등록 확인: MediaCodecList에서 Dolby 디코더 조회
  • Offload 동작: AC-3/E-AC-3 콘텐츠가 HAL에서 직접 디코딩되는지 확인
  • DAP Effects: Dolby Atmos 모드(Movie, Music, Game) 전환 시 청감 변화 확인
  • 패스스루: HDMI/S-PDIF 연결 시 비트스트림이 그대로 전달되는지 확인
  • 채널 매핑: 5.1/7.1 콘텐츠의 스피커 출력 위치 확인

6.2 ADB 디버깅 명령어

# 1. 지원 코덱 확인 adb shell dumpsys media.audio_flinger | grep -i dolby # 2. Audio Policy 상태 adb shell dumpsys media.audio_policy | grep -i "format\|dolby" # 3. HAL 서비스 상태 adb shell dumpsys android.hardware.audio.service # 4. 현재 재생 스트림 정보 adb shell dumpsys media.audio_flinger | grep -A 20 "Output thread" # 5. 7 트러블슈팅

7.1 일반적인 이슈

증상 원인 해결책
소리 없음 MS12 라이브러리 로딩 실패 SELinux 정책 확인, 라이브러리 경로 확인

채널 뒤바뀜 채널 매핑 오류 MS12 출력 채널 순서 설정 확인
DAP 효과 없음 Effect 플러그인 미로딩 audio_effects.xml 및 postprocess 설정 확인
패스스루 실패 HDMI EDID 파싱 오류 HDMI 연결 상태 및 Sink 지원 포맷 확인

7.3 성능 최적화

권장 버퍼 설정

Offload 모드: 32KB ~ 64KB (저지연 불필요 시)
Low-latency 모드: 4KB ~ 8KB (게임 등)
Pass-through: 2KB ~ 4KB (실시간 스트리밍)
