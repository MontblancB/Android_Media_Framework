⚙️ Codec 2.0

차세대 Android 미디어 코덱 아키텍처

1 Overview: Codec 2.0이란?

Codec 2.0은 Android Q(10)에서 도입된 차세대 멀티미디어 프레임워크로, 기존 ACodec/OpenMAX를 대체합니다.

🎯 주요 목표

성능 및 보안 향상

  • Zero-Copy 버퍼 관리로 성능 향상
  • HAL 기반 보안 강화
  • Mainline 모듈로 독립 업데이트
  • 하드웨어 가속 최적화

🔄 기존 시스템 대체

ACodec + OpenMAX IL 대체

  • 더 단순하고 유연한 API
  • 컴포넌트 체이닝 지원
  • 필터 플러그인 지원
  • 구성 쿼리 기능 향상

⚡ Zero-Copy 버퍼

핵심 성능 최적화

  • 데이터 복사 오버헤드 제거
  • C2Buffer를 통한 효율적 전달
  • C2Allocator로 메모리 관리
  • ION/Gralloc 활용

🔌 HAL 인터페이스

android.hardware.media.c2

  • HIDL/AIDL 기반 HAL
  • 벤더 구현 분리
  • 프로세스 간 통신 최적화
  • 독립적인 업데이트 가능

7 Codec 2.0 vs OpenMAX 비교

기존 OpenMAX IL과 Codec 2.0의 주요 차이점을 비교합니다.

🔴 OpenMAX IL (Legacy)

  • 버퍼 관리: 복사 기반, 성능 오버헤드
  • API: 복잡하고 경직된 구조
  • 업데이트: OS 업데이트 필요
  • 확장성: 제한적
  • HAL: 직접 통합, 보안 취약
  • 필터: 지원 제한적

🟢 Codec 2.0 (Modern)

  • 버퍼 관리: Zero-Copy, 고성능
  • API: 단순하고 유연한 구조
  • 업데이트: Mainline 모듈, 독립 업데이트
  • 확장성: 컴포넌트 체이닝, 플러그인
  • HAL: HIDL/AIDL 기반, 보안 강화
  • 필터: 필터 플러그인 지원 (HDR→SDR 등)
⚠️ 마이그레이션 고려사항:
  • Android 10+에서 Codec 2.0이 기본값
  • 기존 OMX 코덱은 호환성 레이어를 통해 계속 동작
  • 새로운 하드웨어 코덱은 Codec 2.0으로 구현 권장
  • 벤더는 C2ComponentStore를 통해 코덱 노출
