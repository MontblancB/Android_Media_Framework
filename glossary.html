📖 Android Media Glossary

미디어 프레임워크 용어 사전 (알파벳 순)

A B C D E F G H I L M N O P R S T U V W

01A

AAC (Advanced Audio Coding)
에이에이씨
MP3의 후속 오디오 코덱으로, 동일 비트레이트에서 더 높은 음질을 제공합니다. MIME 타입: audio/mp4a-latm. Android에서 가장 널리 사용되는 오디오 코덱입니다.
Codec Audio MPEG-4
ABR (Adaptive Bitrate Streaming)
에이비알, 적응형 비트레이트 스트리밍
네트워크 대역폭에 따라 동적으로 비디오 품질을 조정하는 스트리밍 기술입니다. HLS, DASH, SmoothStreaming이 대표적입니다. ExoPlayer의 DefaultTrackSelector가 ABR을 자동으로 처리합니다.
Streaming ExoPlayer HLS DASH
ACodec
에이코덱
Android의 Native 레벨 코덱 래퍼로, OMX (OpenMAX) 코덱을 추상화합니다. Codec 2.0에서는 CCodec으로 대체되었습니다. stagefright 라이브러리의 일부입니다.
Native OMX Codec
AIDL (Android Interface Definition Language)
에이아이디엘
프로세스 간 통신(IPC)을 위한 인터페이스 정의 언어입니다. HAL 3.0+에서 HIDL을 대체하며, MediaCodec, AudioFlinger 등이 AIDL을 사용합니다.
IPC HAL Binder
Audio Focus
오디오 포커스
여러 앱이 동시에 오디오를 재생할 때 우선순위를 관리하는 Android 시스템입니다. AudioManager.requestAudioFocus()로 요청하며, 다른 앱이 포커스를 획득하면 AUDIOFOCUS_LOSS 콜백을 받습니다.
AudioManager Ducking Policy
AudioFlinger
오디오플링어
Android의 Native 오디오 서버로, 모든 오디오 스트림을 믹싱하여 HAL로 전달합니다. AudioTrack, AudioRecord의 백엔드이며, dumpsys media.audio_flinger로 상태를 확인할 수 있습니다.
Native Audio HAL Mixing
AVC (Advanced Video Coding)
에이브이씨
H.264의 공식 명칭입니다. MIME 타입: video/avc. 가장 널리 사용되는 비디오 코덱으로, Android의 모든 디바이스가 지원합니다.
H.264 Codec Video

02B

Binder
바인더
Android의 프로세스 간 통신(IPC) 메커니즘입니다. Framework와 Native Service 간 통신에 사용되며, dumpsys, service list 명령으로 Binder 서비스를 확인할 수 있습니다.
IPC Kernel AIDL
Buffer
버퍼
미디어 데이터를 임시로 저장하는 메모리 영역입니다. MediaCodec의 ByteBuffer, ExoPlayer의 LoadControl이 버퍼를 관리합니다. 버퍼링(Buffering)은 재생 전 충분한 데이터를 미리 로드하는 과정입니다.
MediaCodec ExoPlayer Memory

03C

CCodec
씨코덱
Codec 2.0의 클라이언트 래퍼입니다. ACodec(OMX)을 대체하며, IComponentStore를 통해 Codec 2.0 HAL과 통신합니다. Android 10+에서 기본 코덱입니다.
Codec 2.0 HAL Native
CDD (Compatibility Definition Document)
씨디디, 호환성 정의 문서
Android 디바이스가 준수해야 할 정책 요구사항을 정의한 문서입니다. CTS 테스트의 근거이며, [C-X-Y] 형식의 Requirement ID를 사용합니다. 예: [C-1-1] = Handheld MUST support H.264 1080p@30fps.
CTS Compatibility Policy
Codec
코덱
Coder-Decoder의 약자로, 미디어 데이터를 인코딩/디코딩하는 소프트웨어 또는 하드웨어입니다. H.264, H.265(비디오), AAC, Opus(오디오) 등이 있습니다.
H.264 AAC MediaCodec
Codec 2.0
코덱 투포인트오
Android 9+의 차세대 코덱 HAL입니다. OMX를 대체하며, C++17 기반의 모던 API를 제공합니다. C2Component, C2Work 구조를 사용합니다.
HAL CCodec Native
CTS (Compatibility Test Suite)
씨티에스
Android 호환성 검증 테스트 스위트입니다. CtsMediaTestCases는 MediaCodec, MediaPlayer, DRM 등을 테스트합니다. cts-tradefed 하니스로 실행합니다.
Testing CDD VTS

04D

DASH (Dynamic Adaptive Streaming over HTTP)
대시
MPEG에서 표준화한 ABR 스트리밍 프로토콜입니다. Manifest 파일(.mpd)로 세그먼트 정보를 제공합니다. ExoPlayer의 DashMediaSource로 재생합니다.
Streaming ABR ExoPlayer
Demuxer
디먹서
컨테이너 파일(MP4, MKV)에서 비디오/오디오 스트림을 분리(demultiplexing)하는 컴포넌트입니다. Android의 MediaExtractor가 demuxer 역할을 합니다.
MediaExtractor Container Muxer
DRM (Digital Rights Management)
디알엠, 디지털 저작권 관리
디지털 콘텐츠의 불법 복제를 방지하는 기술입니다. Android는 Widevine, PlayReady, ClearKey를 지원합니다. MediaDrm API로 라이선스를 관리합니다.
Widevine MediaDrm Security
Ducking
더킹
오디오 포커스를 잃었을 때 볼륨을 줄이는(duck) 대신 계속 재생하는 동작입니다. AUDIOFOCUS_LOSS_TRANSIENT_CAN_DUCK 이벤트에서 볼륨을 20%로 낮춥니다. 내비게이션 TTS가 대표적인 예입니다.
Audio Focus AudioManager AAOS

05E

ExoPlayer
엑소플레이어
Google이 개발한 오픈소스 미디어 플레이어 라이브러리입니다. MediaPlayer보다 유연하며, DASH/HLS, DRM, ABR을 지원합니다. 현재는 androidx.media3로 마이그레이션되었습니다.
Media3 Player AndroidX

06F

Frame Drop
프레임 드롭
비디오 재생 중 특정 프레임을 스킵하는 현상입니다. CPU/GPU 오버로드, 버퍼 부족, 디코딩 지연 등이 원인입니다. dumpsys SurfaceFlinger로 모니터링할 수 있습니다.
Performance SurfaceFlinger Rendering
ftrace
에프트레이스
Linux 커널의 이벤트 트레이싱 도구입니다. perfetto, systrace의 백엔드이며, /sys/kernel/debug/tracing에서 직접 제어할 수 있습니다.
Kernel perfetto Debugging

07H

H.264 (AVC)
에이치이육사
가장 널리 사용되는 비디오 코덱입니다. MIME 타입: video/avc. Baseline/Main/High Profile을 지원하며, Android CDD에서 모든 디바이스의 필수 지원 코덱입니다.
Codec AVC MPEG-4
H.265 (HEVC)
에이치이육오
H.264의 후속 코덱으로, 동일 화질에서 50% 적은 비트레이트를 사용합니다. MIME 타입: video/hevc. 4K/8K 비디오에 주로 사용됩니다.
Codec HEVC 4K
HAL (Hardware Abstraction Layer)
할, 하드웨어 추상화 계층
Android Framework와 하드웨어 드라이버 사이의 인터페이스 계층입니다. Audio HAL, Codec HAL 등이 있으며, HIDL/AIDL로 정의됩니다.
HIDL AIDL Vendor
HDCP (High-bandwidth Digital Content Protection)
에이치디씨피
HDMI/DisplayPort의 콘텐츠 보호 기술입니다. Widevine L1 콘텐츠는 외부 디스플레이 출력 시 HDCP 2.2 이상을 요구합니다.
DRM HDMI Widevine
HIDL (Hardware Interface Definition Language)
히들
Android 8.0+의 HAL 인터페이스 정의 언어입니다. Codec 2.0, Audio HAL 6.0 등이 HIDL을 사용합니다. Android 11+에서는 AIDL로 대체되고 있습니다.
HAL AIDL Treble
HLS (HTTP Live Streaming)
에이치엘에스
Apple이 개발한 ABR 스트리밍 프로토콜입니다. Manifest 파일(.m3u8)로 세그먼트를 제공합니다. ExoPlayer의 HlsMediaSource로 재생합니다.
Streaming ABR Apple

08I

I-Frame (Intra-coded Frame)
아이프레임, 키프레임
비디오에서 독립적으로 디코딩 가능한 프레임입니다. Seek 동작은 가장 가까운 I-Frame으로 이동합니다. GOP(Group of Pictures)의 시작 프레임입니다.
Codec Seek GOP
IPC (Inter-Process Communication)
아이피씨, 프로세스 간 통신
Android에서 서로 다른 프로세스 간 데이터를 교환하는 메커니즘입니다. Binder, Socket, Pipe 등이 있으며, 미디어 프레임워크는 주로 Binder를 사용합니다.
Binder AIDL HIDL

09L

Latency
레이턴시, 지연시간
오디오 입력부터 출력까지의 시간 지연입니다. Android CDD는 Automotive 디바이스에 ≤100ms, Handheld에 ≤80ms를 요구합니다. Low-latency 모드는 AudioTrack.PERFORMANCE_MODE_LOW_LATENCY로 활성화합니다.
Audio Performance CDD

10M

Media3
미디어쓰리
ExoPlayer 2.x의 AndroidX 마이그레이션 버전입니다. 패키지명이 androidx.media3.*로 변경되었으며, MediaSession 통합 등 새로운 기능이 추가되었습니다.
ExoPlayer AndroidX Migration
MediaCodec
미디어코덱
Android의 저수준 코덱 API입니다. configure(), start(), queueInputBuffer(), dequeueOutputBuffer()로 인코딩/디코딩을 직접 제어합니다.
Codec Framework Native
MediaExtractor
미디어익스트랙터
컨테이너 파일(MP4, MKV)에서 오디오/비디오 트랙을 추출하는 Android API입니다. readSampleData()로 프레임 데이터를 읽습니다.
Demuxer Container Framework
MediaPlayer
미디어플레이어
Android의 고수준 미디어 재생 API입니다. setDataSource(), prepare(), start()로 간단히 재생할 수 있지만, 커스터마이징이 제한적입니다.
Player Framework Legacy
MediaSession
미디어세션
미디어 재생 상태와 메타데이터를 시스템과 공유하는 API입니다. 잠금 화면, Android Auto, Wear OS에서 재생 제어를 가능하게 합니다.
Framework Notification AAOS
MIME Type
마임 타입
미디어 형식을 나타내는 표준 문자열입니다. 예: video/avc (H.264), audio/mp4a-latm (AAC). MediaCodec.createDecoderByType()에서 사용합니다.
Codec Format
Muxer
먹서
여러 스트림을 하나의 컨테이너 파일로 합치는(multiplexing) 컴포넌트입니다. Android의 MediaMuxer가 muxer 역할을 합니다.
MediaMuxer Container Demuxer

11O

OMX (OpenMAX)
오엠엑스
멀티미디어 코덱의 표준 인터페이스입니다. Android 8 이하에서 코덱 HAL로 사용되었으며, Android 9+에서는 Codec 2.0으로 대체되었습니다.
Legacy Codec 2.0 HAL

12P

perfetto
퍼페토
Android 10+의 시스템 트레이싱 도구입니다. CPU, GPU, Memory, Binder IPC를 추적하며, ui.perfetto.dev 웹 UI에서 분석합니다.
Debugging Profiling ftrace
Performance Class
퍼포먼스 클래스
Android 12+에서 도입된 미디어 성능 등급입니다. Class 11/12/13/14로 구분되며, 각 등급마다 코덱 성능, RAM, 디스플레이 요구사항이 다릅니다. Build.VERSION.MEDIA_PERFORMANCE_CLASS로 확인합니다.
CDD Performance Android 12+

13R

Renderer
렌더러
ExoPlayer에서 미디어를 렌더링하는 컴포넌트입니다. MediaCodecVideoRenderer, MediaCodecAudioRenderer 등이 있으며, 커스텀 렌더러를 구현할 수 있습니다.
ExoPlayer Rendering

14S

SPS/PPS (Sequence/Picture Parameter Set)
에스피에스/피피에스
H.264/H.265 비디오 스트림의 설정 데이터입니다. 해상도, 프로파일, 컬러 포맷 등을 포함하며, MediaCodec의 MediaFormat.KEY_CSD_0로 전달됩니다.
H.264 Codec Configuration
SurfaceFlinger
서피스플링어
Android의 화면 합성(compositing) 서비스입니다. 모든 Surface를 합쳐서 디스플레이로 전송합니다. dumpsys SurfaceFlinger로 프레임 드롭, Vsync 등을 확인할 수 있습니다.
Graphics Native Rendering

15T

Track
트랙
미디어 컨테이너 내의 개별 스트림입니다. 비디오 트랙, 오디오 트랙, 자막 트랙 등이 있으며, MediaExtractor.selectTrack()로 선택합니다.
MediaExtractor Container Stream
TrackSelector
트랙셀렉터
ExoPlayer에서 재생할 트랙을 선택하는 컴포넌트입니다. DefaultTrackSelector는 ABR, 언어, 화질 우선순위를 자동으로 관리합니다.
ExoPlayer ABR Track

16U

Underrun
언더런
오디오 버퍼가 비어서 재생이 끊기는 현상입니다. dumpsys media.audio_flinger의 underruns 카운터로 확인할 수 있습니다. CPU 부족, 버퍼 크기 부족이 원인입니다.
Audio AudioFlinger Performance

17V

VTS (Vendor Test Suite)
브이티에스
HAL 구현을 검증하는 테스트 스위트입니다. Codec HAL, Audio HAL 등을 테스트하며, vts-tradefed 하니스로 실행합니다.
Testing HAL CTS

18W

Widevine
와이드바인
Google의 DRM 솔루션입니다. L1(하드웨어 TEE), L3(소프트웨어) 두 가지 보안 레벨이 있습니다. MediaDrm API로 라이선스를 요청합니다.
DRM Security Google
