미디어 스택 포팅 체크리스트

새로운 SoC/플랫폼으로 Android 미디어 스택을 포팅하는 단계별 가이드

01포팅 개요

새로운 하드웨어 플랫폼에 Android 미디어 스택을 포팅하는 것은 체계적인 접근이 필요합니다. 이 가이드는 SoC 벤더와 OEM 엔지니어가 미디어 포팅을 성공적으로 완료할 수 있도록 단계별 체크리스트를 제공합니다.

02Phase 1: 사전 준비

1
요구사항 분석 및 환경 구축
예상 소요: 1-2주

1.1 하드웨어 요구사항 확인

  • HW 코덱 IP 스펙 확인 (지원 코덱, 해상도, 프로파일)
  • 오디오 DSP 사양 확인
  • 메모리 아키텍처 확인 (ION/DMA-BUF 지원)
  • 보안 하드웨어 확인 (TrustZone, TEE)
  • 디스플레이 출력 사양 확인

1.2 소프트웨어 요구사항

  • 타겟 Android 버전 결정
  • 필수 CDD 요구사항 검토
  • GMS 인증 요구사항 확인
  • 성능 목표 설정 (해상도, FPS, 지연시간)

1.3 개발 환경 구축

  • AOSP 소스 다운로드 및 빌드 환경 구축
  • Vendor 소스 트리 구조 설계
  • 빌드 시스템 설정 (Android.bp/mk)
  • CI/CD 파이프라인 구축

산출물

│       └── sepolicy/

03Phase 2: 코덱 HAL 구현

2
Codec2 HAL 구현
예상 소요: 3-4주

2.1 기본 구조 구현

  • C2ComponentStore 구현
  • C2Component 기본 클래스 구현
  • 버퍼 관리 (C2Buffer, C2Block) 구현
  • HAL 서비스 데몬 작성

2.2 비디오 디코더 구현

  • H.264/AVC 디코더
  • H.265/HEVC 디코더
  • VP9 디코더
  • AV1 디코더 (선택)
  • Secure 디코더 (DRM용)

2.3 비디오 인코더 구현

  • H.264/AVC 인코더
  • H.265/HEVC 인코더
  • VP9 인코더 (선택)

2.4 오디오 코덱 구현

  • AAC 디코더/인코더
  • MP3 디코더
  • Opus 디코더
  • FLAC 디코더

2.5 설정 파일

  • media_codecs.xml 작성
  • media_codecs_performance.xml 작성
  • VINTF manifest 등록

media_codecs.xml 템플릿

<?xml version="1.0" encoding="utf-8"?>
<MediaCodecs>
    <Settings>
        <Setting name="max-video-encoder-input-buffers" value="9" />
    </Settings>

    <Decoders>
        <!-- Video Decoders -->
        <MediaCodec name="c2.vendor.avc.decoder" type="video/avc" rank="0">
            <Limit name="size" min="16x16" max="4096x2160" />
            <Limit name="alignment" value="2x2" />
            <Limit name="block-size" value="16x16" />
            <Limit name="blocks-per-second" min="1" max="2073600" />
            <Limit name="bitrate" range="1-120000000" />
            <Limit name="concurrent-instances" max="16" />
            <Feature name="adaptive-playback" />
            <Feature name="secure-playback" required="true" />
        </MediaCodec>

        <MediaCodec name="c2.vendor.hevc.decoder" type="video/hevc" rank="0">
            <Limit name="size" min="16x16" max="8192x4320" />
            <Limit name="bitrate" range="1-160000000" />
            <Feature name="adaptive-playback" />
        </MediaCodec>
    </Decoders>

    <Encoders>
        <MediaCodec name="c2.vendor.avc.encoder" type="video/avc" rank="0">
            <Limit name="size" min="16x16" max="4096x2160" />
            <Limit name="bitrate" range="1-80000000" />
            <Feature name="bitrate-modes" value="VBR,CBR,CQ" />
        </MediaCodec>
    </Encoders>
</MediaCodecs>

  • USB 오디오 지원
  • HDMI/ARC 오디오 출력

audio_policy_configuration.xml 템플릿

<?xml version="1.0" encoding="UTF-8"?>
<audioPolicyConfiguration version="7.0">
    <modules>
        <module name="primary" halVersion="3.0">
            <mixPorts>
                <mixPort name="primary output"
                         role="source"
                         flags="AUDIO_OUTPUT_FLAG_PRIMARY">
                    <profile name="" format="AUDIO_FORMAT_PCM_16_BIT"
                             samplingRates="48000"
                             channelMasks="AUDIO_CHANNEL_OUT_STEREO"/>
                </mixPort>
            </mixPorts>
            <devicePorts>
                <devicePort tagName="Speaker"
                            type="AUDIO_DEVICE_OUT_SPEAKER"
                            role="sink">
                    <profile name="" format="AUDIO_FORMAT_PCM_16_BIT"
                             samplingRates="48000"
                             channelMasks="AUDIO_CHANNEL_OUT_STEREO"/>
                </devicePort>
            </devicePorts>
            <routes>
                <route type="mix" sink="Speaker"
                       sources="primary output"/>
            </routes>
        </module>
    </modules>
</audioPolicyConfiguration>

06Phase 5: 통합 테스트

5
시스템 통합 테스트
예상 소요: 1-2주

5.1 기능 테스트

  • 기본 재생 테스트 (로컬 파일)
  • 스트리밍 재생 테스트 (HLS, DASH)
  • DRM 콘텐츠 재생 테스트
  • 녹화 테스트
  • 동시 재생 테스트

5.2 성능 테스트

  • 시작 지연 시간 측정 (< 500ms)
  • 프레임 드롭률 측정 (< 0.1%)
  • CPU 사용률 측정
  • 메모리 사용량 측정
  • 배터리 소모량 측정

5.3 안정성 테스트

  • 장시간 재생 테스트 (8시간+)
  • 반복 재생/정지 테스트
  • 메모리 누수 테스트
  • 스트레스 테스트

테스트 자동화 스크립트

#!/bin/bash
# media_stress_test.sh

ITERATIONS=1000
PASS=0
FAIL=0

for i in $(seq 1 $ITERATIONS); do
    echo "=== Iteration $i / $ITERATIONS ==="

    # 재생 테스트
    adb shell am start -a android.intent.action.VIEW \
        -d "file:///sdcard/test_video.mp4" \
        -t "video/*"
    sleep 5

    # 재생 상태 확인
    STATE=$(adb shell dumpsys media.player | grep "state")
    if [[ $STATE == *"PLAYING"* ]]; then
        ((PASS++))
    else
        ((FAIL++))
        adb shell bugreport > "bugreport_$i.zip"
    fi

    # 정리
    adb shell am force-stop com.android.gallery3d
    sleep 2
done

echo "=== Results ==="
echo "Pass: $PASS / $ITERATIONS"
echo "Fail: $FAIL / $ITERATIONS"
