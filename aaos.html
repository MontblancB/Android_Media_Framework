🚗 Android Automotive OS

차량 인포테인먼트를 위한 Android 플랫폼

1 AAOS 개요

Android Automotive OS는 차량 인포테인먼트 시스템(IVI)을 위해 특별히 설계된 Android 플랫폼입니다.

AAOS란? 일반 AOSP에 차량 특화 레이어를 추가하여 ECU, CAN 버스 등 차량 하드웨어와 직접 통신할 수 있는 완전한 차량용 운영체제입니다.

🎯 주요 목적

  • 차량 인포테인먼트 시스템(IVI) 구동
  • 차량 데이터 접근 및 제어
  • 운전자 안전을 위한 UI/UX
  • 멀티 유저/멀티존 지원

🏭 주요 제조사

  • Volvo: Polestar, XC40 Recharge
  • GM: Chevrolet, GMC, Cadillac
  • Ford: Lincoln 시리즈
  • Honda: Acura 모델
  • Renault: Megane E-Tech

⚙️ 핵심 차별점

  • Vehicle HAL (VHAL) 추가
  • Car Services 레이어
  • android.car API 패키지
  • 운전자 주의 산만 방지 가이드라인
  • Car UI Library 템플릿

3 Car Services

Car Services는 앱과 차량 하드웨어 사이의 중재자로, 차량 기능을 Android API로 노출합니다.

🎯 주요 역할

  • 중재자: 앱과 차량 데이터 중개
  • 보안: 권한 기반 접근 제어
  • 추상화: 하드웨어 복잡도 숨김
  • 상호운용성: 제조사 간 호환성

📦 핵심 서비스

  • CarPropertyManager: 차량 속성 관리
  • CarAudioService: 멀티존 오디오
  • CarPowerService: 전원 관리
  • CarUserService: 멀티 유저
  • CarInputService: 입력 이벤트

🔐 보안 및 권한

  • Automotive 특화 권한 시스템
  • 민감한 차량 데이터 보호
  • 앱별 접근 권한 제어
  • OEM 전용 권한

4 Vehicle HAL (VHAL)

Vehicle Hardware Abstraction Layer는 Android와 차량 하드웨어 사이의 표준 인터페이스입니다.

🔌 VHAL 개념

  • 차량 하드웨어 추상화 계층
  • ECU/CAN 버스와 Android 간 번역기
  • 차량 데이터를 속성(Property)으로 노출
  • AIDL/HIDL 기반 인터페이스

📊 차량 속성(Property)

  • 속도: PERF_VEHICLE_SPEED
  • 연료: FUEL_LEVEL_LOW, FUEL_LEVEL
  • 기어: GEAR_SELECTION
  • HVAC: HVAC_TEMPERATURE_SET
  • 도어: DOOR_LOCK, DOOR_POS

⚙️ 속성 타입

  • Access: READ, WRITE, READ_WRITE
  • Change Mode:
    • STATIC: 변하지 않음
    • ON_CHANGE: 변경 시 알림
    • CONTINUOUS: 주기적 업데이트

5 Multi-User Support

AAOS는 차량 내 여러 탑승자가 독립적인 프로필과 앱을 사용할 수 있도록 멀티 유저를 지원합니다.

👥 사용자 격리

  • 각 사용자별 독립 계정
  • 앱 및 데이터 격리
  • 개인 설정 및 환경
  • 프라이버시 보호

🪑 Occupant Zones

  • 탑승자 구역별 사용자 매핑
  • 디스플레이 할당
  • 오디오 존 연결
  • User ID ↔ Occupant Zone 매핑

🔄 사용자 전환

  • 빠른 사용자 전환
  • Guest 모드 지원
  • 운전자 vs 승객 모드
  • 자동 로그인 옵션

6 Multi-Zone Audio

차량 내 여러 구역에서 독립적인 오디오를 동시에 재생할 수 있는 멀티존 오디오 시스템입니다.

🔊 오디오 존 개념

  • 독립적 오디오 출력 그룹
  • 각 존별 독립 포커스
  • 각 존별 독립 볼륨
  • 예: 운전석 vs 뒷좌석

⚙️ 설정 및 관리

  • car_audio_configuration.xml
  • Volume Groups 정의
  • Output 디바이스 매핑
  • CarAudioService 자동 라우팅

✨ Android 14 기능

  • 동적 존 구성
  • Primary Zone Passenger Cast
  • 승객이 메인 캐빈으로 캐스트
  • 운전자 제어권 유지

7 Car UI Library

운전자 안전을 위해 설계된 템플릿 기반 UI 라이브러리입니다.

⚠️ Driver Distraction Guidelines

  • 주행 중 제한된 인터랙션
  • 몇 초 내 완료 가능한 작업
  • 비디오/게임/긴 텍스트 금지
  • 'Distraction Optimized' 태그 필수

📐 템플릿 기반 UI

  • androidx.car.app 라이브러리
  • MediaTemplate: 미디어 플레이어
  • PaneTemplate: 정보 표시
  • MessageTemplate: 메시지/확인

🎤 Voice-First 디자인

  • Google Assistant 통합
  • 핸즈프리 작업
  • 음성 명령 우선
  • 화면 인터랙션 최소화

🚦 Context-Aware UX

  • CarUxRestrictionsManager
  • 주행 중: 간소화된 UI
  • 정차 시: 풍부한 UI
  • 동적 입력 필드 비활성화

8 AOSP vs AAOS 비교

일반 AOSP와 차량용 AAOS의 주요 차이점을 비교합니다.

항목 AOSP (일반) AAOS (Automotive)
목적 스마트폰, 태블릿, TV 차량 인포테인먼트 (IVI)
HAL Camera, Audio, Sensor HAL + Vehicle HAL (VHAL)
추가 레이어 없음 Car Services Layer
API 패키지 android.* 패키지 + android.car.* 패키지
차량 데이터 접근 불가 속도, 연료, 기어, HVAC 접근
멀티 유저 기본 멀티 유저 Occupant Zones 기반
오디오 단일 출력 Multi-Zone Audio
UI 가이드라인 일반 Material Design Driver Distraction Guidelines
UI 라이브러리 표준 Android UI Car UI Library (템플릿)
통신 Wi-Fi, Cellular, Bluetooth + CAN Bus, Vehicle Network
앱 카테고리 모든 카테고리 제한적 (Navigation, Media, Communication)
보안 표준 권한 + Automotive 특화 권한
핵심 요약: AAOS는 AOSP 위에 Car Services, Vehicle HAL, android.car API를 추가하여 차량 하드웨어와의 완전한 통합을 제공하며, 운전자 안전을 최우선으로 하는 UI/UX 가이드라인을 적용한 차량 전용 플랫폼입니다.
